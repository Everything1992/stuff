HTMX Question

Is it possible to have multiple trigger types in HTMX? For example, suppose that I want to trigger the request when a select element (see the element below) changes or loads, and then display the response in an area of my page with id="result". So, if I have the following select element, how would I use/code the htmx attributes for this scenario? Notice that the POST request is already coded.

Yes, it is possible to have multiple trigger types in HTMX by separating them with a comma within the **hx-trigger** attribute. The value "/api/search" represents the URL of the endpoint on your server that will handle the request.
