GitHub Usage Reflection Report

My Current Use of GitHub

After creating my GitHub account at GitHub.com, I initially used the platform mainly as a simple backup for school assignments and web development projects. I created repositories to store my HTML, CSS, and JavaScript files but didn’t fully utilize GitHub’s more powerful features such as branches, collaboration tools, or professional project organization.

Changes After Learning About Online Presence and Git

After reading the lectures on Online Presence and Git & GitHub, I realized that GitHub is not just a storage site — it's an essential part of a developer’s professional image. Moving forward, I plan to improve my GitHub usage by writing detailed README files, properly naming and organizing my repositories, using version control techniques like branching, and contributing to open-source projects when possible.

The Importance of GitHub

GitHub plays a major role in building a visible, credible online presence. It serves as a portfolio, showing code quality, consistency, and project growth. Future employers, collaborators, and clients often review GitHub profiles before making decisions about internships, jobs, or freelance projects.

Using GitHub in the Workplace

In a professional workplace, GitHub is used daily by development teams to manage codebases, track issues, review pull requests, and collaborate on large projects. Understanding GitHub workflows such as cloning repositories, committing changes, creating branches, and submitting pull requests is essential for effective teamwork and software delivery. Companies expect developers to be comfortable using GitHub not only for individual work but also for cross-team collaboration and project documentation.
